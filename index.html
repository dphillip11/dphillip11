Projects

Newton's Cradle▼

I have opted for an interactive demo to lead my portfolio. All of its geometry is modelled using ray intersections and the reflections are exact, this is suitable for such a reflective scene. The scene is rendered using a raytracing algorithm, which is implemented in GLSL. The scene is rendered in real-time, allowing for an interactive experience. You can control the camera using the arrow keys and zoom in and out using the scroll wheel. You can view my shader code here. You can try the demo in fullscreen here.

The obvious benefit of using shaders, and in this example WebGL, is that it can be loaded directly from an html file and run in the browser, furthermore the size of the image is dictated by client and not file size, in fact, the code necessary to create it is only a few hundred lines of text. I will be learning WebGPU as this seems to be the future of web based graphics APIs.

Game Engine▼

Building a game engine is a real test of organisation! My engine is written in C++ and is underpinned by a data structure that is a combination of a unordered map and a vector. This allows for fast iteration over the objects in the scene and fast lookups, the trade off being removal of elements and probably usability to some extent. This forms the basis of my entity component system. Check out my code here!

What else?
  • Developed a tool for parsing .obj files
  • Implemented classes for managing shaders, models, and textures
  • Implemented resource loading to ensure resources are loaded only once
  • Added transform components with parenting to create hierarchies of game objects
  • Implemented oriented bounding box colliders and sphere colliders
  • Implemented collision detection using the separating axis theorem and surface normal tests
  • Implemented rigidbody components for handling physical interactions, including angular momentum
  • Created a GUI using the ImGUI library for creating and manipulating game objects
  • Continued development and future plans include physically based rendering and scripting

Mandelbrot Visualisation▼

This was my first foray into graphics programming as is something of a rite of passage for many. Although I have used webGL to create a demo in the browser, my work on Mandelbrot visualisation mostly took place on the CPU using Python and C. I implemented a variety of methods for rendering the M-set with varying degrees of efficiency. WebGL is very limited in terms of floating point accuracy which certainly is not the case for many of the other implementations. The blurriness that occurs as you zoom is an artifact of the limited precision of the floating point calculations. You can try the demo in fullscreen here.

C was of course an excellent language for outputting still images and I actually used convolutional edge detection to automate this so that I could generate many images at increasing levels of zoom. Python initially seemed a little lethargic but withe the help of the Numba library and the use of the JIT compiler I was able to fare much better.
